# Mental Gymnastics Product Completion Matrix

**Status:** Verified product baseline  
**Date:** 2026-07-11  
**Authority:** `docs/program/training-program.md` and the two foundation documents

## Completion Definition

Mental Gymnastics is complete only when a practitioner can open the app once per program day, receive one evidence-derived prescription, execute it through a drill-specific runtime, receive an objective result, and have Core apply the documented progression response without Android or App inventing a rule.

A runnable screen, completed timer, generated prompt, or persisted session is not sufficient. Every counted prescription must identify:

1. The trained capacity.
2. The current load and the evidence that permits it.
3. The honesty constraints.
4. The executable standard or practice target.
5. The terminal result and classified failure when applicable.
6. The resulting progression, stabilization, maintenance, restoration, recovery, or no-advancement decision.

## Layer Exit Conditions

### Core

- A catalog lookup returns one executable standard definition for every branch-level pair.
- Every threshold, rubric, and critical constraint is tested against pass, boundary, and representative failure evidence.
- Daily prescription consumption, recovery priority, review cadence, and adaptive one-variable load decisions are deterministic and clock-free.

### Runtime and Content

- Every generated instance supplies sufficient hidden expected data for objective scoring.
- Every drill records the facts named by its standard without asking the user to self-certify correctness.
- Snapshot/restore preserves all scoring-relevant state and cannot create a second daily attempt.

### Persistence

- Daily prescriptions and counted attempts are authoritative local records with stable identifiers and integrity validation.
- Schema migration preserves all existing local data, backup/restore includes the new records, and duplicate date consumption is rejected atomically.

### App

- Selection resolves the actual program date, not the first attractive weekly row.
- Completion commits result, evidence, progression transition, adaptive-load history, and daily consumption atomically.
- A consumed date returns a done state; a resumable attempt returns resume; setup cancellation returns the same prescription.

### Android

- Train has exactly one primary daily command before work and no start command after the dose is consumed.
- Every drill offers the inputs required to create objective evidence.
- Result distinguishes practice quality, formal gate outcome, stabilization, ownership, maintenance, decay, transfer, recovery, and failure.
- All curriculum states remain legible on phone, narrow phone, landscape, tablet, and large text.
