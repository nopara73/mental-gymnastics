# First User Experience Workflow

These screenshots show a clean first-user path through the Android app:

1. `01-first-opened-today.png` - first opened Today screen.
2. `02-prescribed-work.png` - current prescribed work.
3. `03-session-preflight.png` - standard, constraint, load, and evidence gate before work starts.
4. `04-live-prep.png` - prepared live session state.
5. `05-live-work.png` - active live session capture for the first Target Hold drill.
6. `06-result-abandoned.png` - abandoned session result with failure evidence and unchanged progress.
7. `07-return-next-action.png` - returned next action after the result.

`firsttimeuser-workflow-contact-sheet.png` is a two-column overview for quick visual review.

Note: the first clean Target Hold session remains labeled `Prep` while the runtime timer runs. Android is rendering the runtime state directly; renaming or splitting that state should happen in Runtime/App rather than in the UI layer.
